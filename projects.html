Projects

Team Fortress 2 Scoreboard November 2014 - December 2014

Team Fortress 2 only allows a limited number of spectators per game. For the web development class, our team created an utility to let many spectators watch a game. We created a plugin which sends out post requests that are picked up by our locally run node server. The server then uses socket.io to send information to clients. The clients maintain state using React.js and Flux. An example game of bots and scoreboard are shown in the video to the right.

Worldvote September 2014 - December 2014

Have you ever wanted to vote a person up or down? That's what worldvote allows you to do. Built with Django, my_sql, and hosted on heroku, our team of three spent the semester creating a site meant to be like reddit but for people for our databases course. Link to github repo. Check out the video on the left for me giving a tour of the site.

Hack Illinois 2015 February 2015

I partnered with a friend and two strangers to create an iOS application, Buzzed Buddy. The application let users find the nearest cheap food, find their way home with the school's bus system, or call a pre-programmed number. To create this, I learned Swift and used it to make HTTP calls to the bus system's API and parsed the JSON response. In addition, I worked with autolayout and constraints to make parts of the UI. Finally, I made the code that stored a friend's phone number and later called it.

Hack Illinois 2014 April 2014

Three classmates and I built an Android application called "Hack Health" to take medical terms as queries and output information such as uses, side effects, name brands, etc. to the user. This was implemented using an API provided by IMO. The API was only available for the weekend of the Hackathon. My role in the project was designing the layout of the app, as well as writing some java helper functions, particularly regarding parsing the information. We were awarded "Best Healthcare Hack", and each team member was awarded an iPad mini.
